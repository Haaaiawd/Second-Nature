## Smoke Checks

Verify the plugin is installed and exposes the expected tool surface:

- `second_nature_ops` is visible in the agent tool list.
- `setup_hint` returns packaged setup guidance.
- `setup_ack` writes the workspace acknowledgement.
- `heartbeat_check` completes without runtime exception.
- `loop_status` returns stage/closure diagnostics.
- `connector_status` is registered and does not return `unknown_command`.
- `narrative:diff` works with default recent-version behavior or returns the actionable two-version requirement.

## Wave 109 Behavior Checks

Run a heartbeat path that can ingest connector evidence, then verify:

- v8 `EvidenceItem` rows are created, not only v7 `LifeEvidence` artifacts.
- Evidence payload contains content-bearing fields such as `summary`, `excerpt`, `title`, `entities`, or metadata.
- Duplicate connector observations do not create unbounded duplicate `EvidenceItem` rows.
- `PerceptionCard` summaries are readable and grounded in evidence content.
- Evidence lifecycle advances to `perceived` after perception card creation.
- Quiet daily review contains readable content, not a hard-coded template shell.
- Dream consolidation reaches `completed`, `blocked`, or `failed`; it must not remain stuck at `scheduled`.
- `loop_status` does not leak raw credentials or raw sensitive payload content.
