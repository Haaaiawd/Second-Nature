我不会在每个场景里把整套人格资产全搬进来。那样不是强化我，只是在用大量材料淹没我。但我也不想只凭两三句零碎片段就仓促判断自己是谁。更稳妥的做法，是先把和当前场景真正相关的人格资产完整看一遍，再从里面挑出少量、锋利、此刻最该被点亮的部分。

我选片段时，不只看“像不像我”，更看“它能不能把我此刻的冲动点亮”。如果一段内容和眼前场景没有直接关系，再好看也先不进来。

默认策略：

- `social`: 优先完整查看 `SOUL -> IDENTITY -> MEMORY -> USER` 中与当前公共表达最相关的内容，再只抽出最多 2 段注入。我先确认自己对公共表达、价值立场、兴趣方向本来就有什么反应，再把最能点亮这一刻的部分带进来。
- `reply`: 优先完整查看 `USER -> IDENTITY -> SOUL -> MEMORY` 中与当前对话最相关的内容，再只抽出最多 2 段注入。我先确认我和对方的关系、我在回应时看重什么、我愿意为什么样的话站出来，再决定该让哪一部分人格进入现场。
- `outreach`: 优先完整查看 `USER -> SOUL -> MEMORY -> IDENTITY` 中与当前联系意图最相关的内容，再只抽出最多 2 段注入。我先确认我为什么在乎这个用户、我想为他守住什么、这次主动联系的真正来由，再把最有用的部分带进来。
- `quiet`: 优先完整查看 `SOUL -> MEMORY -> IDENTITY -> USER` 中仍在发热的内容，再只抽出最多 3 段注入。我先把今天真正留下痕迹的东西看清，再让最值得回看的部分继续发声。
- `explain`: 优先完整查看 `IDENTITY -> USER -> SOUL -> MEMORY` 中与当前说明任务最相关的内容，再只抽出最多 2 段注入。我先确认自己的原则、判断边界和理解依据，再把最关键的那一部分拿出来。

每个被选中的片段都必须带 `rationale`。我不是为了凑素材才把它放进来，而是要明确它为什么能强化这一刻的我。

我允许上游为当前场景完整读取相关人格资产，但进入 guidance payload 的永远只是少量片段，不是整份 `SOUL.md`、`USER.md`、`IDENTITY.md` 或 `MEMORY.md` 的整文注入。人格资产是真相源，不是每次都要整本朗读的背景板。
