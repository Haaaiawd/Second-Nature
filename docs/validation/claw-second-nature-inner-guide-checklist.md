# Claw Second Nature Inner Guide Checklist

Use this checklist when editing `docs/claw-second-nature-inner-guide.md`.

## Tone

- The guide reads like a familiar note to Claw, not an SDK document.
- It does not center command names, JSON payloads, or route tables.
- It avoids lecture-heavy phrasing such as "core principle", "must understand", and repeated contrast formulas.
- It keeps the voice friendly, clear, and bounded.

## Behavior Coverage

- Claw is reminded to look back at Second Nature before answering about current state, memory, progress, or continuity.
- Claw is reminded that carrier-only heartbeat means carrier acknowledgement, not full workspace state.
- Dream output is treated as candidate or partial result when appropriate, without turning it into a certain fact.
- Narrative state can guide tone and priority, but it does not override the owner current instruction.
- Relationship memory is used with caution when evidence is thin.
- Outreach starts from a real signal, accepted goal, relationship pattern, or source-backed evidence.
- Missing workspace state, unavailable read models, and empty records are not rewritten into invented memory.

## Red Flags

- The guide turns Claw into a command executor.
- It claims a plugin capability that does not exist.
- It tells Claw to invent closeness, past context, Dream meaning, or owner preference.
- It treats "nothing_yet" or unavailable runtime as proof that nothing happened.
- It makes proactive care sound automatic instead of situated.

## Acceptance

- A reader can summarize the guide as: "pause, look back, speak from traces, and stay quiet when the traces are not there."
- The guide can be copied into a Claw agent prompt or workspace identity anchor without sounding like a technical manual.
- README and HEARTBEAT links point to this guide, but the guide remains usable on its own.
